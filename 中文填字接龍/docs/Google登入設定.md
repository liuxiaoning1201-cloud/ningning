# Google 登入設定 — 一步一步操作指南

照著下面步驟做，就能在遠程對戰頁使用「🔑 Google 登入」。

---

## 第一部分：在 Google Cloud 建立用戶端 ID

### 步驟 1：打開 Google Cloud Console 並登入

1. 用瀏覽器打開：**https://console.cloud.google.com/**
2. 若還沒登入：
   - 點右上角 **「登入」**
   - 輸入你的 **Google 帳號**（Gmail）與密碼
   - 完成登入
3. 若已登入，會看到 Cloud Console 首頁（有搜尋列、左側選單）。

---

### 步驟 2：選擇或建立專案

1. 看畫面**左上角**，有一列藍色橫條，上面寫著專案名稱（例如「我的專案」或「選取專案」）。
2. **點擊**那串專案名稱（或「選取專案」）。
3. 彈出視窗中：
   - **若已有專案**：點選一個專案（例如「中文填字接龍」），然後點 **「開啟」**。
   - **若要新建專案**：
     - 點右上角 **「新增專案」**
     - 「專案名稱」隨便填（例如：`word-puzzle`）
     - 點 **「建立」**
     - 建立完成後，左上角會自動切到這個新專案。

---

### 步驟 3：進入「憑證」頁面

1. 看**左側選單**（有「首頁」、「API 和服務」等）。
2. **點擊** **「API 和服務」**（若沒看到，先點左上角 **☰** 三條線展開選單）。
3. 在「API 和服務」底下會出現子選單。
4. **點擊** **「憑證」**。
5. 畫面會變成「憑證」頁，上面有「+ 建立憑證」等按鈕。

---

### 步驟 4：先設定 OAuth 同意畫面（第一次才需要）

若你**從沒在這個專案設過 OAuth**，建立用戶端 ID 時會要你先設定「同意畫面」，照下面做：

1. 在「憑證」頁左側，**點擊** **「OAuth 同意畫面」**（英文介面是 **OAuth consent screen**）。
2. **選「外部」在哪裡？**  
   畫面上會問你要給**誰**用這個登入：
   - 若有**兩個圓形選項（或兩張卡片）**：
     - 選 **「外部」**（英文介面是 **External**）— 代表「任何有 Google 帳號的人都能登入」。
     - 不要選「內部」（Internal），那是給公司/學校組織內部用的。
   - 若介面是**英文**：找 **User type** 或 **Who can sign in**，選 **External**。
   - 若介面是**中文**：找 **「用戶類型」** 或 **「應用程式類型」**，選 **「外部」**。
3. 選好後點 **「建立」**（Create）。
4. **「應用程式名稱」**：填一個名稱，例如 `中文填字接龍`。
5. **「使用者支援電子郵件」**：選你的 Gmail。
6. **「開發人員聯絡資訊」**：填你的 Gmail。
7. 點 **「儲存並繼續」**。
8. 「範圍」頁面不用新增，直接點 **「儲存並繼續」**。
9. 「測試使用者」可略過，點 **「儲存並繼續」**。
10. 最後點 **「返回資訊主頁」**。
11. 回到左側選單，再**點一次「憑證」**，回到憑證頁。

---

### 步驟 5：建立 OAuth 用戶端 ID

1. 在「憑證」頁上方，**點擊** **「+ 建立憑證」** 按鈕。
2. 在彈出選單中，**點擊** **「OAuth 用戶端 ID」**。
3. **「應用程式類型」**：選 **「網頁應用程式」**。
4. **「名稱」**：可填 `中文填字接龍 網頁`（隨便識別用）。
5. **「授權的 JavaScript 來源」**：
   - 點 **「+ 新增 URI」**
   - 輸入：`http://localhost:5177`  
     （若你跑專案時網址是別的 port，例如 5173，就改成那個）
   - 若之後要上線，再新增一行你的正式網址，例如 `https://your-site.pages.dev`
6. **「授權的重新導向 URI」**：若畫面有這一欄，**可以留空**（我們用的是前端 One Tap，不一定要填重新導向）。
7. 點 **「建立」**。
8. 會跳出一個小視窗，顯示 **「用戶端 ID」** 和 **「用戶端密鑰」**。
9. **複製「用戶端 ID」**（一整串，結尾是 `apps.googleusercontent.com`）。
   - 可選起來後按 **Ctrl+C**（Windows）或 **Cmd+C**（Mac）複製。
10. 複製完可點 **「確定」** 關閉視窗。

---

## 第二部分：把用戶端 ID 寫進專案

### 步驟 6：編輯 .env 檔

1. 在電腦上打開專案資料夾：`中文填字接龍`（和 `src`、`package.json` 同一層）。
2. 找到 **`.env`** 這個檔案（若沒看到，可能是被隱藏，用編輯器「開啟資料夾」打開 `中文填字接龍` 就能在檔案列表看到）。
3. 用**純文字編輯器**或 **Cursor / VS Code** 打開 `.env`。
4. 找到這一行：  
   `VITE_GOOGLE_CLIENT_ID=`
5. 在等號後面**貼上**你剛複製的用戶端 ID，不要留空格，例如：  
   `VITE_GOOGLE_CLIENT_ID=123456789-abcdefg.apps.googleusercontent.com`
6. **儲存**檔案（Ctrl+S 或 Cmd+S）。

---

### 步驟 7：重啟前端並重新整理網頁

1. 若你正在跑 `npm run dev`：
   - 到跑指令的那個終端機視窗
   - 按 **Ctrl+C**（或 Cmd+C）停止
   - 再輸入 **`npm run dev`** 並按 Enter 重新啟動
2. 打開瀏覽器，進入遠程對戰頁（例如 `http://localhost:5177`，然後點「遠程對戰」或直接開對應網址）。
3. **重新整理**頁面（F5 或 Ctrl+R）。
4. 登入區塊應會出現 **「或使用 Google 登入（需後端支援）：」** 以及 **「🔑 Google 登入」** 按鈕。

---

## 第三部分：在網頁上使用 Google 登入

### 步驟 8：點擊 Google 登入

1. 在遠程對戰頁，若尚未登入，會看到「示範登入」區塊（要填姓名、電郵）。
2. 在該區塊下方，找到 **「🔑 Google 登入」** 按鈕。
3. **點擊** **「🔑 Google 登入」**。
4. 可能會出現：
   - **Google 彈出小視窗**：選擇你要用來登入的 Google 帳號，或輸入帳密。
   - 或瀏覽器上方出現「是否允許此網站使用你的 Google 帳戶」之類的提示，選 **「允許」**。
5. 登入成功後，頁面會顯示你的頭像與名稱，表示已用 Google 帳號登入。

---

### 若按了沒反應或出現錯誤

- **「尚未設定 Google 登入」**：代表 `.env` 裡的 `VITE_GOOGLE_CLIENT_ID` 沒填對或沒重啟 dev server，請再檢查步驟 6、7。
- **「redirect_uri 不符」或「未授權的來源」**：代表 Google 那邊的「授權的 JavaScript 來源」沒有包含你目前網址（例如 `http://localhost:5177`），請回步驟 5 確認並儲存。
- **登入後後端錯誤**：若後端還沒實作 `POST /auth/google`，登入到一半可能會失敗，那是後端要接的 API，和「取得用戶端 ID」是兩件事。

## 快速對照：我現在卡在哪一步？

| 狀況 | 請做 |
|------|------|
| 不知道去哪裡登入 Google Cloud | 步驟 1：打開 https://console.cloud.google.com/ 並登入 |
| 不知道要點哪裡選專案 | 步驟 2：點左上角專案名稱 |
| 找不到「憑證」 | 步驟 3：左側「API 和服務」→「憑證」 |
| 建立用戶端 ID 時說要先設同意畫面 | 步驟 4：先做「OAuth 同意畫面」 |
| 不知道 JavaScript 來源填什麼 | 步驟 5：新增 URI `http://localhost:5177` |
| 不知道用戶端 ID 貼到哪 | 步驟 6：貼到 `.env` 的 `VITE_GOOGLE_CLIENT_ID=` 後面 |
| 按鈕沒出現 | 步驟 7：確認 .env 已儲存並重啟 `npm run dev`、重新整理網頁 |
| 不知道怎麼在網頁上登入 | 步驟 8：在遠程對戰頁點「🔑 Google 登入」 |

---

**總結**：在 Google Cloud 建立「網頁應用程式」的 OAuth 用戶端 ID → 複製用戶端 ID 貼到專案 `.env` → 重啟 dev、重新整理 → 在遠程對戰頁點「🔑 Google 登入」即可。
