# 中文填字遊戲系統 — 技術計劃書（Vue 3）

## 一、專案概述

### 1.1 目標

打造一套**中文填字遊戲系統**，幫助學生掌握**中文字詞、成語與語句**（如論語等經典），並支援**練習、本地對戰與遠程班級對戰**。教師可依教學進度隨時新增或修改題庫，系統支援 **Excel / CSV** 匯入、**自動排版**與**難度分級**，並參考「論語填字接龍卡」的題目版／解答版格式進行設計。

### 1.2 參考格式說明（論語填字接龍卡）

依據您提供的 PDF（題目版／解答版），格式要點如下：

| 項目 | 說明 |
|------|------|
| **網格** | 行列可標示為 A/B/C…（橫向）與 一/二/三…（豎向），與提示對應 |
| **格子類型** | **提示字（已填）**：預先顯示的字；**待填空格**：學生需填寫，可標題號 |
| **橫向提示** | 如「A. 君子__器 《為政》」，對應該行待填字與出處 |
| **豎向提示** | 如「一. 知為知 《為政》」，對應該列待填字與出處 |
| **難度** | 以星級表示（★☆☆☆☆～★★★★★），並可配關卡名稱（如「第一關：初學入門」） |
| **出處** | 可標註來源，如《為政》《學而》《里仁》等，方便教學對應 |

系統需支援：**橫排、豎排皆可作答**，且同一格可能同時屬於一橫一豎兩條提示（交叉填字）。

## 二、技術棧（Vue 3）

| 類別 | 技術選型 | 說明 |
|------|----------|------|
| **前端框架** | Vue 3 (Composition API) | 單頁應用、組件化、響應式 |
| **建構工具** | Vite | 開發體驗與產物優化 |
| **語言** | TypeScript | 型別安全、資料結構清晰 |
| **路由** | Vue Router 4 | 練習／對戰／教師後台等路由 |
| **狀態管理** | Pinia | 全域狀態（用戶、房間、題目、設定） |
| **樣式** | CSS 變數 + 可選 Tailwind / UnoCSS | 主題與響應式 |
| **遠程對戰** | WebSocket（如 Socket.io 或原生 WS） | 即時同步、房間與回合 |
| **後端（遠程對戰）** | Node.js（Express/Fastify）或 Nuxt 3 全端 | 班級、房間、題目派發、成績 |
| **認證（遠程）** | 學校郵箱登入（OAuth2 / 自建驗證） | 加入班級、身分識別 |
| **資料匯入** | 前端解析 Excel（xlsx 庫）/ CSV（papaparse 或手寫） | 教師匯入題庫、自動排版與難度分級 |

## 三、功能規劃

### 3.1 教師端

- **題庫管理**
  - 新增／編輯／刪除「字詞」「成語」「語句」題庫。
  - 支援 **Excel 或 CSV** 匯入，欄位可約定例如：`詞句, 出處, 難度, 備註`；系統依難度與字數做**自動排版**與**難度分級**（見下節）。
- **填字題編輯**
  - 手動編輯網格：設定「提示字（已填）」與「待填空格」、題號、橫向／豎向提示文字與出處。
  - 可從題庫選擇詞句，由系統**自動生成**交叉網格（演算法需考慮橫豎交叉、重複字）。
- **難度與關卡**
  - 為每題或題組設定星級難度與關卡名稱（如「第一關：初學入門」）。
  - 匯入時可依欄位「難度」自動分級或歸入不同關卡。
- **題組與班級**
  - 題組可綁定至「班級」或「公開」，供練習或遠程對戰使用。
  - 遠程對戰時教師可選擇題組與時間限制，派題給班級。

### 3.2 Excel / CSV 匯入與自動化

- **約定格式（示例）**
  - CSV：`詞句, 出處, 難度, 備註`  
    例：`知為知之, 《為政》, 1,`、`君子不器, 《為政》, 2,`
  - Excel：同一 sheet 多行，欄位同上；可支援多 sheet 對應不同單元。
- **自動排版**
  - 依「詞句」長度與可交叉字（共同字）生成二維網格，保證橫向、豎向皆可對應到提示。
  - 可設定：最小/最大網格尺寸、是否允許「僅橫」或「僅豎」的詞條。
- **難度分級**
  - 依匯入欄位「難度」或字數、出處數量等規則，自動標記星級或關卡，供遊戲內「依難度出題」使用。

### 3.3 遊戲設定（學生端／通用）

- **難度選擇**：在設定中選擇難度（如 1～5 星或關卡），系統依此**自動出題**（從題組中篩選）。
- **提示開關**：可選「顯示提示」或「不顯示提示」（僅顯示空格數或題號），以配合練習或測驗。
- **橫排／豎排**：同一局內橫向與豎向皆可作答，介面上需標示當前為「橫向題」或「豎向題」並對應題號（參考論語填字接龍卡之 A/B/C 與 一/二/三）。

### 3.4 三種遊戲模式

| 模式 | 說明 | 技術要點 |
|------|------|----------|
| **練習模式** | 單人自選題組或依難度自動出題，可開關提示，本地儲存進度與錯題 | 本地儲存（localStorage / IndexedDB）、可選同步到後端 |
| **本地對戰** | 兩人同一裝置輪流或同時作答，比正確率或完成時間 | 本地狀態（Pinia），無需後端 |
| **遠程對戰** | 學生以**學校郵箱登入**，加入**班級**，多人同時在各自電腦上作答同一題組，即時排名或回合結束後公布成績 | 後端 API + WebSocket、班級與房間、認證、題目派發、結果彙整 |

### 3.5 遠程對戰細項

- **登入**：學校郵箱（如 `@school.edu.tw`）登入，驗證方式可為：OAuth2（Google/Microsoft 等）限定網域，或自建帳密+信箱驗證。
- **班級**：教師建立班級並分享班級代碼／連結；學生加入後可看到該班級的對戰活動。
- **房間**：教師或系統建立「對戰房間」，選擇題組、難度、時間限制與是否顯示提示；學生進入房間後同時開始作答。
- **即時同步**：WebSocket 推送開始/結束、倒數、即時完成人數或排名（可選），並在結束時寫入成績供教師查詢。

## 四、資料模型（概念）

- **題庫**
  - 詞條：`id, 詞句, 出處, 難度, 備註`（與 CSV/Excel 對齊）。
- **填字題**
  - 網格：二維陣列，每格為「提示字」或「待填（題號）」；可額外標註該格所屬橫向/豎向提示 ID。
  - 橫向提示列表：`id, 題號/標籤, 提示文字, 出處, 答案字串`。
  - 豎向提示列表：同上。
  - 難度、關卡名稱、題組 ID。
- **題組**
  - `id, 標題, 關卡/難度範圍, 題目列表, 建立者, 班級可見性`。
- **用戶／班級／房間**
  - 用戶：`id, 學校郵箱, 顯示名稱, 班級列表`。
  - 班級：`id, 名稱, 代碼, 教師, 學生列表`。
  - 房間：`id, 題組, 設定（時間、提示）, 參與者, 開始/結束時間, 成績快照`。

（實際實作時可再細化為 REST/GraphQL 與資料庫 schema。）

## 五、系統架構概覽

```
┌─────────────────────────────────────────────────────────────────┐
│                         Vue 3 SPA (Vite)                         │
├──────────────┬──────────────┬──────────────┬─────────────────────┤
│  教師後台     │  練習模式     │  本地對戰     │  遠程對戰（學生端）   │
│  題庫/題組    │  選題/設定    │  雙人本地     │  登入/班級/房間      │
│  Excel/CSV   │  難度/提示    │  同一裝置     │  即時作答/排名       │
└──────┬───────┴──────┬───────┴──────┬───────┴──────────┬──────────┘
       │              │              │                   │
       ▼              ▼              ▼                   ▼
┌──────────────┐  ┌──────────────┐  (無)           ┌─────────────────┐
│ localStorage │  │ localStorage │                  │  Backend API    │
│ / 題組 JSON  │  │ / 進度與設定  │                  │  + WebSocket    │
└──────────────┘  └──────────────┘                  │  班級/房間/成績  │
                                                    └─────────────────┘
```

- **練習 / 本地對戰**：可僅用前端 + 本地儲存，題組可匯入/匯出 JSON 或檔案。
- **遠程對戰**：需後端提供 REST（班級、房間、題目、成績）與 WebSocket（即時事件）。

## 七、實作階段建議

| 階段 | 內容 | 產出 |
|------|------|------|
| **Phase 1** | Vue 3 + Vite + Router + Pinia 專案；資料型別與本地儲存；參考論語填字接龍卡格式的**單題填字 UI**（橫/豎提示、提示字與待填、題號） | 可手動組一題並完成練習模式單題作答與對答案 |
| **Phase 2** | 教師端：題庫 CRUD、**Excel/CSV 匯入**、**自動排版**與**難度分級**；題組管理；遊戲設定（難度、提示開關）與**依難度自動出題** | 教師可匯入題庫並生成題組，學生可選難度與提示練習 |
| **Phase 3** | **本地對戰**：雙人輪流或同時、計分/計時與結果畫面 | 本地雙人可玩 |
| **Phase 4** | **遠程後端**：學校郵箱登入、班級、房間、題目派發、成績儲存；**WebSocket** 即時同步 | 可建立班級與房間 |
| **Phase 5** | **遠程對戰前端**：登入、加入班級、進入房間、同時作答、即時排名或結束後成績 | 完整遠程多人對戰 |

## 八、風險與注意事項

- **自動排版**：中文交叉填字需考慮「共同字」與網格密度，演算法可能需多輪迭代或半自動（系統建議、教師微調）。
- **學校郵箱**：須與校方或既有 SSO 協定（如 OAuth2 限定網域），或自建驗證流程並處理個資與合規。
- **即時同步**：WebSocket 需處理斷線重連、遲到加入、作弊防護（如僅送交卷結果、不即時送答案）。

## 九、總結

本計劃書以 **Vue 3** 為技術棧，涵蓋：

- **參考論語填字接龍卡**的題目/解答格式，支援橫排豎排交叉作答與提示/出處。
- **教師**可隨時新增修改題庫，並透過 **Excel/CSV** 匯入達成**自動排版**與**難度分級**。
- **遊戲設定**包含難度選擇、系統自動出題及是否顯示提示。
- **三種模式**：練習、本地對戰、遠程對戰（學校郵箱、班級、多人同時比賽）。

實作可依上列階段分步進行，先完成練習與教師端與本地對戰，再接入遠程後端與 WebSocket，逐步擴充至完整班級對戰功能。
