# 📜 「詩友記」—— 穿越時空的詩人聊天軟件項目書

> **讓經典走進對話，讓詩人成為朋友**

---

## 一、項目概述

### 1.1 項目名稱
**「詩友記」（PoetPal）**

### 1.2 項目定位
一款以 AI 驅動的沉浸式古典文學社交學習應用，讓中小學生透過仿社交媒體（WeChat / Instagram / QQ）的聊天介面，與中國歷代詩人、文學家進行「穿越時空」的對話，從而提升文言文閱讀能力、激發主動學習興趣。

### 1.3 目標用戶
| 學習階段 | 年級 | 核心需求 |
|---------|------|---------|
| 第一學習階段 | 小一至小三 | 啟蒙認識、趣味互動、圖文並茂 |
| 第二學習階段 | 小四至小六 | 故事探索、詩詞鑒賞入門、角色收集 |
| 第三學習階段 | 中一至中三 | 深度對話、文學脈絡理解、創作練習 |
| 第四學習階段 | 中四至中六 | 賞析思辨、考試備考、跨文本比較 |

### 1.4 核心理念
- **以聊代讀**：將傳統閱讀轉化為即時對話體驗
- **以友代師**：詩人不是遙不可及的古人，而是你的「好友」
- **以玩代學**：遊戲化機制讓學習自然發生

### 3.4 功能模組總覽

#### 模組一：私聊對話 💬
- **自由聊天**：與詩人就任意話題展開對話，AI 以該詩人的語氣、性格、知識背景回應
- **語音訊息**：詩人可「朗誦」詩詞（TTS 配古風配樂）
- **圖片分享**：AI 生成水墨風格場景圖配合對話情境
- **表情包**：古風 Q 版詩人表情包（如李白舉杯、杜甫嘆氣、蘇軾吃東坡肉）
- **「翻譯模式」**：一鍵切換文言 ↔ 白話對照

#### 模組二：詩人朋友圈 / 動態 📰
- 詩人定期發佈「動態」（結合其生平事件、代表作品）
- 學生可點讚、留言、轉發
- 觸發「隱藏劇情」—— 例如李白和杜甫互相評論
- **節氣/節日聯動**：清明時杜牧發《清明》、中秋時蘇軾發《水調歌頭》、重陽時王維發《九月九日憶山東兄弟》

#### 模組三：群聊功能 👥
- **「詩人茶聚」**：將不同朝代的詩人拉入同一群聊
  - 例：「豪放派群聊」—— 李白 + 蘇軾 + 辛棄疾
  - 例：「田園詩人群聊」—— 陶潛 + 王維 + 孟浩然
  - 例：「憂國憂民群聊」—— 杜甫 + 范仲淹 + 岳飛
- 觀察不同詩人之間的「對話」，體會文風差異
- 學生可提問，多位詩人從不同角度回應

### 4.2 角色 AI 設計原則

每位詩人的 AI 角色需具備：

1. **性格一致性**：語氣、用詞、態度始終符合歷史人物形象
2. **知識準確性**：所引用的詩詞、歷史事件必須準確無誤
3. **教育導向性**：自然地引導學生深入理解作品，而非單純聊天
4. **適齡調節**：根據學習階段自動調整對話深度和用語難度
5. **互動引導**：主動提問、出題、推薦延伸閱讀，保持學生參與度

### 4.3 示範對話（李白 × 第二階段學生）

> **學生**：李白你好！你為什麼這麼喜歡月亮？
>
> **李白**：哈哈，小友好！你問得好。月亮嘛，她是我最忠實的朋友！不管我走到哪裏——長安的酒樓、廬山的瀑布旁、還是異鄉的客棧——抬頭一看，她都在。
>
> 我有一首詩叫《靜夜思》，就是一個睡不著的夜晚，看到床前灑滿月光，一下子就想起家鄉了。你讀過嗎？
>
> **學生**：讀過！床前明月光，疑是地上霜。
>
> **李白**：好！接下去呢？ 😄
>
> **學生**：舉頭望明月，低頭思故鄉。
>
> **李白**：👏 厲害！全對了！告訴你一個秘密——我寫這首詩的時候才二十幾歲，離開家鄉已經好幾年了。你有沒有試過離開家人，特別想念他們的時候？
>
> 💡 *[系統提示：你已成功背誦《靜夜思》！親密度 +10，解鎖「月下詩人」徽章]*

### 4.4 示範對話（蘇軾 × 第四階段學生）

> **學生**：蘇軾先生，《水調歌頭》中「但願人長久，千里共嬋娟」是寫給誰的？
>
> **蘇軾**：這首詞是寫給我的弟弟子由的。那年中秋，我在密州做太守，已經七年沒和他見面了。你想想，七年！月圓人不圓，心裏實在難受。
>
> 不過我沒有一直傷感下去。我轉念一想——月有陰晴圓缺，人有悲歡離合——這本就是天地間的常態，何必執著呢？只要彼此都健康平安，即使相隔千里，看的也是同一輪明月，不也是一種「團圓」嗎？
>
> 你覺得，蘇某這樣想是「豁達」還是「自我安慰」？ 🤔
>
> **學生**：我覺得是豁達。因為現實不能改變，改變心態才是最好的。
>
> **蘇軾**：說得好！人生不如意事十常八九，能轉換心境，便能活得自在。這也是我一生被貶了那麼多次，還能寫出這些詩詞的原因。對了，你知道我被貶到黃州的時候發明了什麼嗎？ 😏
>
> 💡 *[快捷按鈕：「東坡肉的故事」/「被貶經歷」/「繼續賞析」]*

## 六、遊戲化與激勵機制

### 6.1 親密度系統

每位詩人有獨立的「親密度」等級：

| 等級 | 名稱 | 所需親密度 | 解鎖內容 |
|------|------|----------|---------|
| Lv.1 | 萍水相逢 | 0 | 基礎對話 |
| Lv.2 | 點頭之交 | 100 | 詩人背景故事 |
| Lv.3 | 把酒言歡 | 300 | 隱藏詩作 + 專屬表情包 |
| Lv.4 | 知音難覓 | 600 | 深度話題解鎖（政治觀、人生觀） |
| Lv.5 | 莫逆之交 | 1000 | 詩人「真心話」+ 限定頭像框 |

### 6.2 文士等級體系

| 等級 | 稱號 | 特權 |
|------|------|------|
| 1-5 | 蒙童 | 基本功能 |
| 6-10 | 書生 | 解鎖群聊功能 |
| 11-15 | 秀才 | 可參加「科舉」挑戰 |
| 16-20 | 舉人 | 解鎖所有詩人 |
| 21-25 | 進士 | 可發佈原創作品 |
| 26-30 | 翰林 | 專屬書房裝飾 + 全功能開放 |

### 6.3 每日驚喜機制

- **「晨間詩語」**：每日早晨推送一句詩詞 + 詩人問候
- **「詩人來訪」**：隨機詩人主動發起對話（如：「杜甫想和你聊聊天」）
- **「詩禮紅包」**：回答詩詞問題即可拆開，獲得金幣或裝飾品
- **「穿越日記」**：每日查看一則詩人的「日記」，了解其生平

## 八、技術方案

### 8.1 技術選型

| 層面 | 技術選擇 | 說明 |
|------|---------|------|
| 前端框架 | Flutter / React Native | 跨平台（iOS + Android + Web） |
| 後端框架 | Node.js (NestJS) / Python (FastAPI) | 高併發、易擴展 |
| AI 模型 | GPT-4 / Claude API + 微調模型 | 角色扮演 + 知識問答 |
| 知識檢索 | RAG (LangChain + 向量數據庫) | 確保回答的文學準確性 |
| 語音合成 | Azure TTS / 訊飛 TTS | 古風朗誦效果 |
| 圖像生成 | Stable Diffusion / DALL-E | 水墨風格場景圖 |
| 數據庫 | PostgreSQL + Redis + Pinecone | 結構化數據 + 緩存 + 向量存儲 |
| 即時通訊 | WebSocket (Socket.io) | 模擬即時聊天體驗 |
| 雲服務 | AWS / 阿里雲 / 騰訊雲 | 按地區選擇最優方案 |

### 8.2 數據安全與隱私

- 遵循《個人資料(私隱)條例》（香港）
- 所有學生數據加密存儲
- 對話內容安全過濾（防止不當內容）
- 未成年人保護機制
- 定期安全審計
